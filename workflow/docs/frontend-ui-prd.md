## Executive Summary

前端 Web UI 是 AI Agent 工作流引擎的双入口之一（另一入口为 DSL/CLI），面向**业务运营人员**与**平台开发者**提供可视化的工作流编排、监控与运维能力。核心包括基于 ReactFlow 的画布编辑器（拖拽搭建 DAG 工作流）、任务执行监控面板（DAG 拓扑 + 节点详情 + 时间轴）、以及工作流模板市场，三者通过统一的 REST API 与后端引擎交互。

本 PRD 聚焦 **Phase 3（第 5-6 周）** 交付范围，采用 React 18 + TypeScript + Vite + ReactFlow + Ant Design 技术栈，以 Docker Compose 容器化方式与后端一同部署。

## Problem Statement

**Current Situation**: 目前引擎仅支持 YAML DSL 与 CLI 两种交互方式，业务运营人员无法直接使用；此外缺乏可视化的任务执行回溯能力，排查问题效率低。

**Proposed Solution**: 构建一套完整的 React 前端应用，提供：

- 可视化画布编辑器，支持拖拽节点、连线、配置参数
- 任务监控面板，展示 DAG 拓扑执行状态与节点级详情
- 工作流模板库，开箱即用降低上手成本
- DSL ↔ 画布双向同步，开发者与运营者共享同一份工作流定义
- 基础用户管理与 API Key 鉴权

**Business Impact**:
- 业务运营人员可零代码搭建工作流，覆盖范围扩大 3x+
- 任务排查效率提升，失败定位从分钟级降到秒级
- 模板市场降低新用户上手时间至 15 分钟内

## User Personas

### Primary: 业务运营人员（Business Operator）
- **Role**: 产品经理、数据分析师、运营人员
- **Goals**: 不写代码，通过画布拖拽快速搭建「取数 → AI 总结 → 发通知」类工作流
- **Pain Points**: 现有方案要么纯代码门槛高，要么纯界面功能弱
- **Technical Level**: 低代码友好型（熟悉 Excel / Notion，不写 Python）
- **使用页面**: 模板市场、画布编辑器、任务列表、任务详情

### Secondary: 平台开发者（Developer）
- **Role**: 后端/平台工程师
- **Goals**: 通过 Web UI 快速验证 DSL 正确性，监控线上任务执行状态
- **Technical Level**: 中高级，熟悉 YAML/DAG 概念
- **使用页面**: 画布编辑器（DSL 模式）、任务详情页、设置页

### Tertiary: 管理员（Admin）
- **Role**: 系统管理员
- **Goals**: 管理用户、API Key、查看系统运行状态
- **Technical Level**: 中级
- **使用页面**: 登录页、设置页、任务列表

## User Stories & Acceptance Criteria

### Story 1: 画布编辑器 — 拖拽搭建工作流

**As a** 业务运营人员
**I want to** 从左侧节点面板拖拽节点到画布，通过连线表达依赖关系，并配置每个节点的参数
**So that** 我能不写代码搭建出一个可运行的工作流

**Acceptance Criteria:**
- [ ] 左侧面板展示 8 类核心节点（LLM / Tool / Condition / Loop / Retriever / Approval / HTTP / SubWorkflow），每个节点有图标和名称
- [ ] 节点可拖拽到画布，画布支持平移（Pan）和缩放（Zoom）
- [ ] 节点之间拖拽连线建立依赖关系，连线方向从上游指向下游
- [ ] 选中节点后右侧面板展示参数配置表单，不同类型的节点渲染不同的配置项
- [ ] 画布支持撤销/重做（Undo/Redo）
- [ ] 画布支持「选中 → 删除」快捷键（Delete/Backspace）
- [ ] 画布支持节点对齐辅助线（Snap-to-grid）
- [ ] 连线规则自动校验（禁止循环依赖、禁止孤立节点）

### Story 2: 画布编辑器 — DSL 双向同步

**As a** 平台开发者
**I want to** 在画布编辑器中切换 DSL 视图，直接编辑 YAML 并实时反映到画布
**So that** 我能利用 DSL 的精确性做精细调整

**Acceptance Criteria:**
- [ ] 画布编辑器提供「可视化」与「DSL」两种 Tab 视图
- [ ] DSL 视图集成 Monaco Editor 或 CodeMirror，带 YAML 语法高亮和 Schema 校验
- [ ] 切换至 DSL 视图时，画布内容自动序列化为 YAML
- [ ] 在 DSL 视图中修改并点击「应用」后，画布重新渲染，位置信息自动布局
- [ ] DSL 校验失败时在编辑器中标红并显示错误位置
- [ ] 双向同步 100% 等幂：画布 → DSL → 画布 前后内容一致

### Story 3: 工作流保存与触发

**As a** 业务运营人员
**I want to** 保存当前画布中的工作流定义，并一键触发执行
**So that** 我能完成从搭建到运行的完整闭环

**Acceptance Criteria:**
- [ ] 顶部工具栏提供「保存」按钮，保存成功后显示提示并记录版本
- [ ] 保存后的工作流出现在工作流列表中
- [ ] 工具栏提供「触发执行」按钮，点击后弹出输入参数对话框（JSON 编辑器）
- [ ] 触发成功后自动跳转至任务详情页
- [ ] 未保存时触发执行，自动先保存再执行

### Story 4: 工作流列表页

**As a** 平台开发者
**I want to** 查看所有已保存的工作流，支持搜索、筛选和管理操作
**So that** 我能快速找到目标工作流进行编辑、触发或删除

**Acceptance Criteria:**
- [ ] 列表展示：工作流名称、节点数量、最近修改时间、状态（已发布/草稿）
- [ ] 支持按名称搜索和按状态筛选
- [ ] 每行提供操作按钮：编辑、触发执行、导出 YAML、复制、删除
- [ ] 顶部提供「新建工作流」按钮，点击跳转至空画布
- [ ] 支持「导入 YAML」按钮，上传后自动校验并跳转到画布编辑器
- [ ] 空状态展示引导提示（含 3 个模板入口）

### Story 5: 任务列表页

**As a** 平台开发者 / 业务运营人员
**I want to** 查看所有历史执行任务，按状态和时间筛选
**So that** 我能快速定位失败任务并排查问题

**Acceptance Criteria:**
- [ ] 列表展示：任务 ID（简短版）、所属工作流名称、执行状态、触发时间、耗时
- [ ] 状态标签带颜色：运行中（蓝色）、成功（绿色）、失败（红色）、等待审批（橙色）
- [ ] 支持按状态、按工作流名称、按时间范围筛选
- [ ] 支持按创建时间倒序排列
- [ ] 每行点击跳转至任务详情页
- [ ] 提供「重跑」操作按钮（失败/完成状态的任务）

### Story 6: 任务详情页 — DAG 拓扑与节点详情

**As a** 平台开发者
**I want to** 在任务详情页中看到完整的 DAG 执行拓扑，每个节点的状态、输入/输出、耗时、日志
**So that** 我能快速定位失败节点并理解失败原因

**Acceptance Criteria:**
- [ ] 顶部概览区：工作流名称、任务 ID、整体状态、执行总耗时、重试次数
- [ ] 中间 DAG 拓扑图：节点按执行状态着色（待执行灰、运行中蓝、成功绿、失败红、跳过黄、等待审批橙）
- [ ] 点击 DAG 中的节点，下方展开节点详情面板：
  - 节点名称和类型
  - 输入参数（JSON 格式化展示）
  - 输出结果（JSON 格式化展示，可折叠）
  - 执行耗时
  - Token 用量（LLM 节点）
  - 错误信息（如有）
  - 重试记录（如有）
- [ ] 提供「从该节点重跑」按钮（针对失败节点）
- [ ] 提供「重跑整个任务」按钮
- [ ] 提供时间轴视图切换（DAG 拓扑 / 时间轴）

### Story 7: 用户登录与 API Key 管理

**As a** 管理员
**I want to** 登录系统并在设置页中管理 API Key
**So that** 系统有一定的基础安全保障

**Acceptance Criteria:**
- [ ] 登录页：用户名 + 密码表单，登录成功后跳转至工作流列表页
- [ ] 登录态保持：token 存储在 localStorage，请求时自动附带
- [ ] 未登录访问任意页面自动跳转至登录页
- [ ] 设置页：查看当前 API Key、生成新 Key（旧 Key 自动失效）、复制到剪贴板
- [ ] 设置页：修改密码功能
- [ ] 退出登录按钮

### Story 8: 模板市场

**As a** 新用户
**I want to** 从模板市场中选择一个预置工作流模板，一键创建
**So that** 我能快速上手体验引擎能力

**Acceptance Criteria:**
- [ ] 模板卡片网格展示，每个卡片包含：模板名称、简短描述、节点数量、预计完成时间
- [ ] 至少提供 3 个模板：每日巡检报表、批量文档处理、自动报表生成
- [ ] 点击「使用模板」按钮，自动创建工作流并跳转至画布编辑器
- [ ] 模板创建成功后画布中已含预置节点和连线，用户可继续编辑
- [ ] 模板本身不可编辑（只读），基于模板创建的工作流可自由修改

## Functional Requirements

### Core Features

#### Feature 1: 画布编辑器（核心页面）

**页面路由**: `/editor/:workflowId`

**布局**:

```
+----------------------------------+-------------------------------------------+
|  [Logo] Agent Workflow              用户头像 [v]                       |
+------------+----------------------------------------------------------+
|            |                                                          |
|  节点面板   |              ReactFlow 画布区域                        |
|            |          (平移 / 缩放 / 对齐辅助线)                      |
|  --------- |                                                          |
|  LLM       |                                                          |
|  Tool      |                                                          |
|  Cond      |                                                          |
|  Loop      |                                                          |
|  RAG       |                                                          |
|  Approve   |                                                          |
|  HTTP      |                                                          |
|  SubFlow   |                                                          |
|            |                                                          |
+------------+----------------------------------------------------------+
|  底部状态栏: 节点数 | 已保存 | 校验状态                         |
+-------------------------------------------------------------+
```

**工具栏**（画布上方）:
- 回退（Undo）/ 重做（Redo）
- 保存 / 另存为
- 触发执行（带输入参数对话框）
- 导出 YAML / 复制到剪贴板
- 视图切换（可视化 / DSL）
- 自动布局（Auto-Layout）
- 全屏模式

**右侧抽屉面板**（选中节点后弹出）:
- 节点名称（可编辑）
- 节点类型（只读）
- 参数配置（类型相关）:
  - LLM: System Prompt（多行文本）、Model（下拉）、Temperature（滑块）、Max Tokens（数字输入）
  - Tool: 选择已注册工具（下拉）、参数映射（JSON）
  - Condition: 条件表达式（输入框）
  - Loop: 迭代列表来源（表达式）、循环变量名、最大迭代次数
  - Retriever: 知识库选择（下拉）、TopK（数字输入）、Score Threshold（滑块）
  - Approval: 审批人（输入）、超时时间、审批消息模板
  - HTTP: URL、Method、Headers、Body
  - SubWorkflow: 工作流选择（下拉）、输入参数映射
- 重试策略（展开子面板）:
  - 最大重试次数
  - 重试间隔（秒）
  - 重试条件（白名单异常类型）

**Error handling**:
- 网络请求失败时显示全局 Toast 提示
- 保存冲突时提示用户刷新
- 校验失败时在画布上标记无效节点（红色边框）+ 错误文字提示

**页面路由**: `/templates`

**布局**: 卡片网格，3 列响应式

3. **自动报表生成**: 查询数据库 → LLM 分析趋势 → 生成图表描述 → 邮件通知

- 实时 WebSocket 推送（仅轮询）
- E2E 测试录制与回放
- 用户注册（仅管理员预创建）

## Technical Constraints

### Performance
| 指标 | 目标值 |
|---|---|
| 首屏加载时间 (LCP) | ≤ 3s（开发机 localhost） |
| 页面切换（路由跳转） | ≤ 500ms |
| 画布渲染 50 节点 | ≥ 30 FPS 拖拽/缩放 |
| API 请求超时 | 10s，超时后显示友好提示 |
| 任务详情轮询间隔 | 3s（运行中）/ 不轮询（已完成） |

### Security
- 登录 token 存 localStorage，请求头 `Authorization: Bearer <token>`
- 路由守卫：未登录自动跳转 `/login`
- API Key 在 UI 中掩码展示（`sk-********`）
- 敏感操作（删除工作流、生成 API Key）需二次确认
- XSS 防护：所有用户输入渲染前做 HTML 转义
- 接口请求统一异常拦截（401 跳登录、403 提示无权限）

### Integration
- **后端 API**: 统一 RESTful 接口，Base URL 通过环境变量 `VITE_API_BASE_URL` 配置
- **认证接口**: `POST /api/auth/login` → 返回 token
- **工作流 CRUD**: `GET/POST/PUT/DELETE /api/workflows[/:id]`
- **任务接口**: `GET /api/runs`（列表）、`GET /api/runs/:id`（详情）、`POST /api/runs`（触发）
- **节点详情**: `GET /api/runs/:id/nodes/:nodeId`（单节点输入输出）

### 核心设计原则

基于竞品分析，确立 5 条设计原则：

| 原则 | 说明 | 来源参考 |
|---|---|---|
| **1. 响应式信息密度** | 画布区域最大化，面板可折叠/可收起，不遮挡主编辑区 | LangFlow、n8n |
| **2. 状态即颜色** | 节点/任务状态通过颜色一目了然，不依赖文字阅读 | Airflow、Prefect |
| **3. 拖拽即创建** | 从侧栏拖出即创建节点，无需先点按钮再选类型 | LangFlow、Flowise |
| **4. 所见即所得** | 画布上的节点参数修改立即反映到 DSL（反之亦然） | Dify |
| **5. 渐进式复杂** | 初始界面简洁（默认隐藏高级选项），用户需要时展开 | n8n |

### 交互模式参考

#### 节点拖拽创建（参考 LangFlow）

```
1. 用户从左侧面板拖拽节点类型
2. 拖拽过程中在画布上显示放置预览（半透明节点）
3. 放下时自动创建节点 + 自动分配 ID（llm_1, tool_2）
4. 新创建节点自动处于选中态，右侧面板打开参数配置
```

#### 节点连线（参考 n8n 出口/入口锚点 + LangFlow 拖拽连线）

```
1. 鼠标悬停在节点时显示出口/入口锚点（圆形，与节点色一致）
2. 从出口锚点拖出，显示贝塞尔曲线预览（虚线）
3. 拖到目标节点入口锚点时高亮 + 吸附
4. 放下后创建连线 + 自动校验（循环依赖检测）
5. 连线成功后添加微动效（涟漪效果）
```

#### DAG 执行状态实时更新（参考 Prefect + Airflow Grid View）

```
1. 任务运行中时前端每 3 秒轮询 /api/runs/:id
2. 节点状态变化时：
   a. 状态色即时切换（灰->蓝->绿/红）
   b. 连线动态：当前活跃边脉冲流动画（虚线光效）
   c. 已完成节点添加对勾角标
   d. 失败节点抖动 + 红色脉冲扩散
3. 轮询到终态后停止，展示完整结果
```

#### DSL 视图切换（参考 Dify 代码/可视化切换）

```
1. 画布顶部 Tab 切换: [可视化] [DSL]
2. 切换到 DSL 标签时:
   a. 画布淡出
   b. Monaco Editor 淡入（带 YAML 语法高亮）
   c. 自动序列化当前画布为 YAML
3. 切换到可视化标签时:
   a. 如果有未应用的 DSL 修改，弹出确认
   b. 确认后解析 YAML 为画布节点（自动布局）
   c. 解析失败时返回错误消息并停留在 DSL 标签
```

### 组件库选型对比

| 选型 | Ant Design 5.x | shadcn/ui | MUI 5 |
|---|---|---|---|
| 企业级组件丰富度 | 丰富（Table/Form/Modal 开箱即用） | 较基础，需自行组合 | 丰富但较重 |
| 主题定制 | Design Token 全面覆盖 | Tailwind 灵活 | 样式覆盖较麻烦 |
| Bundle Size | 较大（需 tree-shaking + 按需加载） | 极小 | 较大 |
| 学习成本 | 文档完善，社区大 | 简单直观 | 较复杂 |
| 适合场景 | 企业后台、复杂表格、表单 | 轻量工具、startup | 中后台 |
| 最终选择 | Ant Design 5.x | | |

**理由**: 项目面向企业级工作流引擎后台，需要丰富的 Table、Form、Modal、Drawer、Notification 等组件；Ant Design 的 Design Token 体系能很好地与品牌色集成；社区生态成熟，参考 Dify 的成功案例。
